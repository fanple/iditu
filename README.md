<a name="license" class="anchor" href="#license"><span class="octicon octicon-link"></span></a>License</h2>

<ul>
<li><a href="http://www.apache.org/licenses/LICENSE-2.0.html">Apache Version 2.0</a></li>
</ul><h2>
<a name="building" class="anchor" href="#building"><span class="octicon octicon-link"></span></a>Building</h2>

<p>The build requires <a href="http://maven.apache.org/download.html">Maven</a>
v3.0.3+ and the <a href="http://developer.android.com/sdk/index.html">Android SDK</a>
to be installed in your development environment. In addition you'll need to set
the <code>ANDROID_HOME</code> environment variable to the location of your SDK:</p>

<pre><code>export ANDROID_HOME=/opt/tools/android-sdk
</code></pre>

<p>After satisfying those requirements, the build is pretty simple:</p>

<ul>
<li>Run <code>mvn clean package</code> from the <code>app</code> directory to build the APK only</li>
<li>Run <code>mvn clean install</code> from the root directory to build the app and also run
the integration tests, this requires a connected Android device or running
emulator</li>
</ul><p>You might find that your device doesn't let you install your build if you
already have the version from the Android Market installed.  This is standard
Android security as it it won't let you directly replace an app that's been
signed with a different key.  Manually uninstall GitHub from your device and
you will then be able to install your own built version.</p>

<p>See <a href="https://github.com/github/android/wiki/Building-From-Eclipse">here</a> for
instructions on building from <a href="http://eclipse.org">Eclipse</a>.</p>
